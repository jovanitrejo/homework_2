The time_checking.py script completes Part IV-A. It takes the two external IPs of the US and EU servers, and then contacts the 2 given endpoints ten times for each server, using the time package to record the time elapsed before the request was successfully processed. Print statements are used to display each time elapsed for each call, and are then saved to calculate the averages for each server at the end. The averages are then printed at the end.

The eventual_nature.py script completes Part IV-B. It uses a misses/errors counter to keep track of how many times the server fails to return data, or when the server returns stale data. Using each ip address for the US and EU servers, a loop of 100 times is executed, and each loop uses the US server instance to register a username to the firestore database. Afterwards, the script then uses the EU server to get the list of all usernames from the firestore database. This immediate read will allow us to see if there are any stale reads from the firestore. Results are displayed for each loop execution via print statements in the terminal. At the end, a summary of how many misses and errors occured during the loop execution.
